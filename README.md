## О проекте

**Карьерная книга** — это корпоративная платформа для управления компетенциями сотрудников. Система автоматизирует процесс оценки навыков, формирования индивидуальных планов развития (ИПР) и подбора учебных материалов.

### Основные возможности

#### Для сотрудника
- **Мои компетенции** — просмотр текущего и целевого уровней владения компетенциями, отслеживание прогресса изучения материалов
- **Индивидуальный план развития (ИПР)** — канбан-доска для отслеживания статуса учебных материалов (Запланировано / В работе / Готово к проверке)
- **Учебные материалы** — доступ к материалам, рекомендованным для текущего уровня компетенций
- **Dashboard** — общая информация о прогрессе развития и последних обновлениях

#### Для руководителя
- **Мой отдел** — список сотрудников с их прогрессом
- **Компетенции отдела** — матрица компетенций всех сотрудников, возможность назначения/удаления компетенций, выставления оценок
- **Учебные материалы отдела** — просмотр материалов, назначенных сотрудникам, с фильтрацией по компетенциям
- **Dashboard руководителя** — статистика по отделу, прогресс сотрудников, последние события

#### Для администратора
- **Управление пользователями** — назначение ролей (Администратор / Руководитель / Сотрудник)
- **Управление компетенциями** — создание и редактирование компетенций, привязка учебных материалов к уровням
- **Управление учебными материалами** — добавление/редактирование материалов с указанием типа и длительности
- **Иерархия должностей** — построение структуры «Должность → Уровни → Компетенции», привязка компетенций к уровням должностей
- **Импорт из Excel** — загрузка матриц компетенций и профилей сотрудников

## Особенности

### Бизнес-логика
1. **Уровни компетенций (0-3)**: 
   - 0 — Не определен
   - 1 — Базовые знания
   - 2 — Профессионал
   - 3 — Эксперт

2. **Прогресс изучения материалов**:
   - Статусы: 0 — Запланировано, 1 — В работе, 2 — Готово к проверке
   - Прогресс вычисляется как процент изученных материалов от всех материалов компетенции

3. **Next-level материалы**:
   - Сотрудник видит только материалы для перехода с текущего уровня на следующий
   - Материалы прошлых уровней не отображаются
   - Эндпоинт: `/api/educational-material-competencies/by-competency/{id}/next-level`

4. **Общие компетенции отдела**:
   - При выборе "Все сотрудники" отображаются только компетенции, которые есть у ВСЕХ сотрудников отдела
   - Можно добавлять/удалять компетенции как отдельному сотруднику, так и всем сразу

### Технические особенности

1. **Drag & Drop**:
   - Перетаскивание материалов между уровнями компетенций
   - Перетаскивание материалов между компетенциями
   - Запрет дубликатов материалов в одной компетенции

2. **Кастомные выпадающие списки**:
   - Стилизованные select с кастомной стрелкой через `background-image`
   - Единый дизайн на всех страницах

3. **CSS переменные**:
   - Цветовая схема через `variables.module.scss`
   - Поддержка светлой/темной/системной темы
   - Адаптивные отступы и размеры

4. **Унифицированные стили**:
   - Единый дизайн страниц: отступы, скругления, тени
   - Все страницы используют `$variables-background` для фона и `$variables-containers-background` для контента

## Ролевая модель

### Администратор
- Полный доступ ко всем функциям системы
- Управление пользователями и их ролями
- Управление компетенциями и учебными материалами
- Управление иерархией должностей
- Импорт матриц компетенций из Excel

### Руководитель
- Просмотр компетенций сотрудников своего подразделения
- Назначение/удаление компетенций сотрудникам
- Просмотр учебных материалов отдела
- Фиксация оценок компетенций
- Просмотр прогресса подчиненных

### Сотрудник
- Просмотр своих компетенций
- Формирование индивидуального плана развития (ИПР)
- Доступ к учебным материалам для следующего уровня
- Просмотр своего прогресса

## Установка и запуск

### Предварительные требования

- Node.js (рекомендуется версия 18.0.0 или выше)
- npm (входит в состав Node.js)
- Бэкенд сервер, запущенный на `http://localhost:5217`

### Клонирование репозитория

```bash
git clone <repository-url>
cd Frontend
```
### Установка зависимостей

```bash
npm install
```
### Запуск приложения в режиме разработки
```bash
npm start
```
Приложение будет доступно по адресу http://localhost:3000
### Сборка для продакшена
```bash
npm run build
```

## API Эндпоинты

### Основные эндпоинты
